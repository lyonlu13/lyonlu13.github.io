名稱中文解釋備註
DMA (Direct Memory Access)直接記憶體存取為避免外接裝置在需要搬移資料到記憶體時過度占用到CPU(大材小用的概念)，允許讓裝置直接存取DRAM。*Intel 的 DDIO 技術 *起始位置通常會被限制
DDIO (Intel® Data Direct I/O Technology)I/O加速技術DMA的一種，可允許外接裝置直接存取CPU的Cache（反正停留在DRAM的時間也不長，早要進Cache），可提高速度，對於網路操作特別有效。
Interrupt 中斷中斷是指處理器接收到外圍硬體或軟體的信號，導致處理器通過一個執行 context switch 並處理該事件。
Dual-Mode避免應用程式隨便執行重大指令，將操作或記憶體空間隔離為User/Kernel兩部分。
User Mode此模式下無法使用特權指令(Privileged Instruction)，且無法隨意操縱記憶體，只可使用Kernel分配的記憶體空間。
Kernel Mode此模式下可使用特權指令(Privileged Instruction)，並可隨意調配記憶體。

ACL (Access-control list))存取控制串列/訪問控制表存取控制串列描述用戶或系統行程對每個物件的存取控制權限。
System Call系統呼叫指運行在使用者空間(User Space)的程式向操作系統核心請求需要更高權限運行的服務。
RIP (Register Instruction Pointer)程式指位器主掌CPU動向的暫存器(放下一個指令位址)*詳看 https://ithelp.ithome.com.tw/m/articles/10227112
IVT (interrupt vector table)中斷向量表紀錄發生中斷時要進行的ISR*最多2048個ISR
ISR (interrupt service routine)中斷處理程式發生中斷時執行的函數，是驅動程式的起始點
Nested Interrupt巢狀中斷一個中斷被另外一個中斷中斷
CFS (Completely Fair Scheduler)完全公平排程器負責將CPU資源，分配給正在執行中的行程，目標在於最大化程式互動效能與整體CPU的使用率。*能優化多媒體與遊戲效能，在延遲方面
inode「類Unix檔案系統」中的一種資料結構，用於描述檔案系統物件。每個inode儲存了檔案系統物件資料的屬性和磁碟塊位置。檔案系統物件屬性包含了各種元資料，也包含使用者群組和權限資料。
SMP (symmetric multiprocessor)對稱多處理器所有運算用的處理器是等價的，換句話說應用程式可以在這些處理器上進行移轉。
CMP (chip multiprocessor)常常稱之為multi-core，與傳統的SMP架構不同的地方在於：通常最下層的快取記憶體（LLC，last level cache）是共享的。因此CMP上不同核心間交換資料較為快速
SMT (Simultaneous multithreading)同步多執行緒增加暫存器的數量來模擬多個CPU，讓多個Task競爭加法器等資源，以達到有效運用效能的目的(系統運用率)。

I/O 的運作方式

🌐 https://mropengate.blogspot.com/2015/01/operating-system-ch2-os-structures.html

三種將資料在 I/O 間傳送的方法：

  • Polling：最簡單的方式讓 I/O 裝置與 CPU 溝通。I/O 裝置只要將資訊放進狀態暫存器，CPU 會周期性的檢查並取得資訊來得知需要服務的裝置。
  • Interrupt-driven I/O：利用 interrupt 的機制，當一個 I/O 裝置需要服務時，會發出 interrupt 來通知 CPU (非同步I/O)。
  • DMA (Direct Memory Access)：提供一個裝置控制器，讓 I/O 裝置能夠直接在記憶體做資料的傳輸，不需要 CPU 的參與。
    • cycle stealing: DMA 向 CPU 竊用機器週期，而直接向記憶體存取資料，因此 CPU 可與週邊裝置的工作同時進行。萬一 CPU 與 DMA 對記憶體存取發生衝突，則給 DMA 較高的優先權 (執行消耗小的先執行)。

DMA 與 CPU 搶記憶體控制權時的處理方式

  • DMA 一次傳小量資料，盡快把控制權還給CPU ⇒ 沒有效率
  • DMA 大量傳輸，提升I/O效能 ⇒ CPU閒置

⇒ 由仲裁器(Arbiter) 決定

ISR 與 Signal 比較

ISR 硬體呼叫 Signal 作業系統呼叫

執行 ISR 前需先把CPU的狀態存下來，以便恢復中斷前的工作。

這點Signal也一樣，但會由作業系統負責處理CPU的狀態保存。

兩者皆做越少事越好，Signal 可在接收訊號時設定對應的flag，等適當時機再執行此事件。ISR則可保留到 Kernel Thread (核心執行緒) 再進行處理。

中斷的軟體處理

  • ISR 可以視情況啟用中斷，代表允許巢狀中斷(Nested Interrupt)
  • 在Linux中，ISR只處理須立即處理的部分，其餘交給 Kernel Thread
  • 中斷可改變Task狀態，中斷結束後，交由Scheduler 決定下個 Task
  • 不能呼叫會造成wait的函數
    • wait的主體是task，而ISR只是函數
    • ISR只處理必要部分
    • 需要wait或大量工作的話就觸發bottom half (由Kernel Thread 呼叫) 丟給softirq(高並行)/tasklet(中)collection-queue(同類task只占用一個執行緒執行可避免race condition)等去執行
      • 會提供函數/資料指標(驅動作者寫的)，softirq依序執行
      • softirq最多僅支援32個驅動，通常只有高速裝置會掛在上面(網路卡/硬碟)
      • 大部分用work-queue就好

Thread 與 Process 皆為 Task

對Linux來說，Thread 與 Process 只差在共用資源(記憶體)的多寡。

在Kernel Mode 下執行的 Thread 為 Kernel Thread
